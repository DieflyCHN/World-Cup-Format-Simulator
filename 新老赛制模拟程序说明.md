# 新老赛制模拟程序说明

本文用于说明本项目中的世界杯赛制模拟程序。

## 文件说明

本仓库核心材料包括四个文件：

```text
current_format_simulator.py：12组×4队、前二加最好第三名赛制模拟程序
new_format_simulator.py：16组×3队、三队2分桥接制模拟程序
新老赛制模拟程序说明.md：中文说明文件
format_simulator_explanation.md：英文说明文件
```

---

## Quick Start

运行 12 组赛制模拟：

```bash
python3 current_format_simulator.py 10000 -w 8 --seed 2026
```

运行桥接赛制模拟：

```bash
python3 new_format_simulator.py 10000 -w 8 --seed 2026
```

参数含义：

```text
10000：模拟10000届世界杯
-w 8：使用8个工作线程
--seed 2026：固定随机种子，便于复现结果
```

---

本文说明两个独立模拟程序：

```text
current_format_simulator.py：12组×4队、前二加最好第三名赛制
new_format_simulator.py：16组×3队、三队2分桥接制
```

两个程序使用相同的核心概率模型、相同的球队 rating、相同的命令行形式和相同的输出字段。差异只来自赛制结构本身。

---

## 一、核心模拟概念说明

### 1. Monte Carlo 模拟

本程序使用 Monte Carlo 模拟。

其基本思想是：

```text
重复随机模拟一届完整世界杯很多次；
记录每支球队在每次模拟中走到哪一阶段；
用次数比例估计出线率、晋级率和夺冠率。
```

例如模拟 10000 次后，若某队 7200 次小组出线，则其出线率记为：

```text
7200 / 10000 = 72.0%
```

程序本身不试图预测某一场真实比赛的唯一结果，而是通过大量随机重复来观察赛制结构下的长期概率分布。

---

### 2. 球队强度：Elo-like rating

每支球队有一个 `rating`，可理解为近似 Elo 强度值。

rating 越高，球队越强。程序中的球队强度大致处于：

```text
1200 - 2200
```

该 rating 不是官方排名，也不是精确预测模型，而是用于模拟中区分强弱的输入参数。

---

### 3. Elo 胜负概率

淘汰赛、点球，以及其他必须分出胜负的场景，使用 rating 直接计算二选一胜率。

公式为：

```text
P(A胜B) = 1 / (1 + 10 ^ ((rating_B - rating_A) / K))
```

程序中：

```text
K = 800
```

含义是：

```text
两队 rating 越接近，胜率越接近 50%-50%；
rating 差距越大，强队胜率越高；
但弱队仍然保留爆冷概率。
```

---

### 4. 单场状态波动

淘汰赛和点球不是只按固定 rating 计算。

程序会给每队加入一场比赛级别的随机状态波动：

```text
match_rating = rating + 随机状态
```

其中随机状态服从正态分布：

```text
均值 = 0
标准差 = 120
```

这表示：

```text
强队通常仍然更强；
但单场比赛存在状态、临场发挥、偶然性。
```

---

### 5. 小组赛比分：泊松模型

小组赛需要具体比分，因为排名需要用到：

```text
积分
净胜球
总进球数
相互比赛结果
```

因此小组赛不直接抽“胜平负”，而是先生成比分。

程序用泊松分布生成进球数。

基本流程为：

```text
先根据两队 rating 差计算双方预期进球；
再根据预期进球随机生成实际进球数；
最终得到一个 90 分钟比分。
```

预期进球的计算方式为：

```text
A预期进球 = base + (rating_A - rating_B) / rating_scale
B预期进球 = base - (rating_A - rating_B) / rating_scale
```

程序中：

```text
base = 1.25
rating_scale = 800
min_goals = 0.20
```

含义是：

```text
两队接近时，双方预期进球都约为 1.25；
强队预期进球更高，弱队预期进球更低；
min_goals 防止弱队预期进球被压到不现实的 0。
```

---

### 6. 点球处理

点球不计入进球数，也不影响净胜球。

在新赛制中，跨组赛如果 90 分钟打平：

```text
90分钟比分保留为平局；
该比分计入双方进球数和净胜球；
再用 Elo 胜负模型模拟点球胜者；
点球胜者获得跨组赛2分。
```

例如：

```text
A 1-1 B
点球 A 胜
```

则积分和数据记为：

```text
A：进球+1，失球+1，净胜球0，跨组赛胜，得2分
B：进球+1，失球+1，净胜球0，跨组赛负，得0分
```

点球不会把比分改成 2-1，也不会给胜者增加净胜球。

---

### 7. 随机种子

两个模拟程序都支持 `--seed` 参数。

随机种子的作用是让同一条命令可以复现同一组模拟结果：

```bash
python3 current_format_simulator.py 10000 -w 8 --seed 2026
python3 new_format_simulator.py 10000 -w 8 --seed 2026
```

在并行模拟时，程序会从基础 seed 派生出每个任务块自己的随机种子。只要以下参数相同：

```text
模拟次数
核心数
seed
```

输出结果即可复现。

## 二、12组赛制模拟程序说明

12组赛制模拟程序为：

```text
current_format_simulator.py
```

它用于模拟 48 队、12 组、每组 4 队、小组前二加最好第三名晋级的赛制。

---

### 1. 程序目标

`current_format_simulator.py` 只做一件事：

```text
在12组赛制下重复模拟完整世界杯，并输出每支球队的阶段胜率。
```

命令行参数为：

```bash
python3 current_format_simulator.py <模拟次数> -w <核心数> --seed <随机种子>
```

例如：

```bash
python3 current_format_simulator.py 10000 -w 8 --seed 2026
```

其中 `--seed` 可省略；省略时每次运行都会使用新的随机结果。

---

### 2. 每次模拟的完整流程

每一次模拟都代表一届完整世界杯。

流程为：

```text
1. 随机抽签，将48队放入A1到L4；
2. 进行12个四队小组的小组阶段；
3. 每组4队进行单循环比赛；
4. 每组前二直接晋级；
5. 12个小组第三横向比较，选出8个最好第三名；
6. 按第三名分配表确定32强对阵；
7. 按固定32强签表进行淘汰赛；
8. 记录每队最终阶段；
9. 重复很多次后输出概率。
```

---

### 3. 随机抽签

程序每次模拟都会重新随机抽签。

48 支球队会被随机洗牌，然后依次填入：

```text
A1、A2、A3、A4、
B1、B2、B3、B4、
……
L1、L2、L3、L4
```

这意味着模拟结果同时包含：

```text
球队实力影响
抽签随机性影响
赛制结构影响
```

程序不使用某一届赛事的固定分组。

---

### 4. 小组结构

老赛制共有 12 个四队小组：

```text
A、B、C、D、E、F、G、H、I、J、K、L
```

每组 4 队。

以 `X组` 为例：

```text
X组：X1、X2、X3、X4
```

小组赛为单循环：

```text
X1-X2
X1-X3
X1-X4
X2-X3
X2-X4
X3-X4
```

因此每组有 6 场比赛，每队踢 3 场小组赛。

---

### 5. 积分规则

小组赛使用普通足球积分：

```text
胜：3分
平：1分
负：0分
```

所有小组赛比分都由泊松模型生成。

---

### 6. 小组排名规则

程序按以下顺序排名：

```text
1. 总积分
2. 总净胜球
3. 总进球数
4. 相互比赛积分
5. 相互比赛净胜球
6. 相互比赛进球数
7. rating
8. 随机兜底值
```

其中第 4 至第 6 项只在前 3 项仍无法区分球队时使用。

程序没有公平竞赛积分数据，因此使用：

```text
rating
随机兜底值
```

作为最后的排序兜底。

---

### 7. 晋级规则

每组四队排名后：

```text
小组第1晋级
小组第2晋级
小组第3进入横向比较
小组第4淘汰
```

12 个小组第三名再按以下顺序横向比较：

```text
1. 总积分
2. 总净胜球
3. 总进球数
4. 胜场数
5. rating
6. 随机兜底值
```

排名最高的 8 个小组第三名晋级。

因此：

```text
12组 × 2队 = 24队
8个最好第三名 = 8队
合计32队晋级
```

---

### 8. 32强签表

32 强使用固定签表。

其中 8 个小组第一会遇到一个最好第三名。具体第三名来自哪个小组，取决于 8 个晋级第三名分别来自哪些小组。

程序中使用第三名分配表处理这一规则：

```text
先确定晋级第三名的小组集合；
再查表确定 A1、B1、D1、E1、G1、I1、K1、L1 分别对阵哪个小组第三。
```

固定 32 强基础签表为：

```text
A2-B2
E1-第三名
F1-C2
C1-F2
I1-第三名
E2-I2
A1-第三名
L1-第三名
D1-第三名
G1-第三名
K2-L2
H1-J2
B1-第三名
J1-H2
K1-第三名
D2-G2
```

程序内部使用比赛编号：

```text
73、74、……、88
```

后续淘汰赛用：

```text
W73、W74、……
```

表示对应比赛的胜者。

这些编号只是程序内部引用，不代表真实比赛编号。

---

### 9. 淘汰赛模拟

淘汰赛从32强开始，全部为单败淘汰。

每场淘汰赛必须产生胜者。

程序使用 Elo 胜负模型，并加入单场状态波动：

```text
match_rating = rating + 随机状态
```

然后计算双方胜率并随机产生胜者。

这等价于把：

```text
90分钟
加时赛
点球
```

合并为一个“最终谁晋级”的结果。

---

### 10. 输出结果

程序输出每支球队在全部模拟中的阶段概率：

```text
出线
16强
8强
4强
决赛
冠军
```

示例列为：

```text
球队    出线    16强    8强    4强    决赛    冠军
```

其中：

```text
出线 = 进入32强
16强 = 通过32强赛
8强 = 通过16强赛
4强 = 通过8强赛
决赛 = 通过半决赛
冠军 = 赢得决赛
```

---

### 11. 程序边界

本程序适合回答的问题是：

```text
在给定球队强度、随机抽签和12组赛制结构下，
各队长期晋级概率大致如何分布？
```

本程序不适合被解释为：

```text
对真实世界杯的精确预测；
对某一场比赛比分的精确预测；
对球队真实实力的官方评估。
```

它的主要用途是：

```text
为12组赛制提供结构性、概率性的模拟参考。
```

`new_format_simulator.py` 只做一件事：

